# UI компоненты (UI)

## Обзор

UI компоненты представляют пользовательский интерфейс приложения PDFmanager. Они обеспечивают взаимодействие с пользователем и отображение данных из приложения.

## Компоненты UI

### Components
- **Назначение**: Переиспользуемые UI компоненты
- **Функции**:
  - Кнопки, поля ввода, переключатели
  - Таблицы и списки
  - Прогресс-бары
  - Всплывающие уведомления
  - Диалоговые окна

### Layout
- **Назначение**: Компоненты макета
- **Функции**:
  - Основные структурные элементы
  - Навигация между режимами
  - Адаптивные сетки
  - Управление видимостью блоков

### Theme
- **Назначение**: Управление темами
- **Функции**:
  - Переключение между светлой/темной темой
  - Управление цветовыми схемами
  - Сохранение предпочтений пользователя
  - Синхронизация тем между окнами

### Modals
- **Назначение**: Модальные окна
- **Функции**:
  - Диалоги подтверждения
  - Настройка параметров
  - Просмотр логов
  - Обратная связь

### Widgets
- **Назначение**: Интерактивные виджеты
- **Функции**:
  - Drag&drop области
  - Миниатюры PDF
  - Прогресс-индикаторы
  - Панели управления

## Архитектурные решения

### Компонентный подход
Каждый UI компонент изолирован и может быть использован независимо от других компонентов.

### Реактивность
Компоненты реагируют на изменения состояния приложения и автоматически обновляют отображение.

### Адаптивность
Все компоненты адаптируются к различным размерам экрана и разрешениям.

### Доступность
Компоненты соответствуют стандартам доступности (WCAG) и поддерживают клавиатурную навигацию.

## Использование

### Создание компонента
```typescript
import { Button } from '@ui/components/button';
import { ThemeManager } from '@ui/theme/theme-manager';

// Создание кнопки
const button = new Button({
  text: 'Нажми меня',
  variant: 'primary',
  onClick: () => console.log('Кнопка нажата')
});

// Применение темы
ThemeManager.applyTheme('dark');
```

### Использование Layout компонентов
```typescript
import { Layout } from '@ui/layout/layout';
import { Navigation } from '@ui/layout/navigation';

const layout = new Layout({
  container: document.getElementById('app'),
  theme: 'dark'
});

const navigation = new Navigation({
  items: [
    { id: 'merge', label: 'Объединение', icon: 'merge-icon' },
    { id: 'compress', label: 'Сжатие', icon: 'compress-icon' },
    { id: 'settings', label: 'Настройки', icon: 'settings-icon' }
  ],
  onNavigate: (id) => {
    layout.showView(id);
  }
});
```

### Работа с модальными окнами
```typescript
import { Modal } from '@ui/modals/modal';
import { ConfirmModal } from '@ui/modals/confirm-modal';

// Простое модальное окно
const modal = new Modal({
  title: 'Заголовок',
  content: '<p>Содержимое модального окна</p>',
  buttons: [
    { text: 'ОК', action: 'confirm' },
    { text: 'Отмена', action: 'cancel' }
  ]
});

// Модальное окно подтверждения
const confirmModal = new ConfirmModal({
  message: 'Вы уверены, что хотите очистить все настройки?',
  onConfirm: () => {
    // Действие при подтверждении
  }
});
```

## Стилизация

### CSS модули
Каждый компонент использует изолированные CSS стили для предотвращения конфликтов.

### Темизация
Стили компонентов адаптируются под выбранную тему с использованием CSS переменных.

### Анимации
Компоненты включают плавные анимации для улучшения пользовательского опыта.

## Тестирование

UI компоненты тестируются с использованием:
- Модульных тестов для логики компонентов
- Интеграционных тестов для взаимодействия
- Визуальных регрессионных тестов

## Доступность

### Клавиатурная навигация
Все интерактивные элементы доступны через клавиатуру с использованием Tab.

### ARIA атрибуты
Компоненты включают соответствующие ARIA атрибуты для скринридеров.

### Контрастность
Цветовые схемы соответствуют требованиям по контрастности.

## Безопасность

- Все пользовательские данные санитизируются перед отображением
- Предотвращение XSS атак через безопасное встраивание HTML
- Ограничение на размер отображаемых данных
